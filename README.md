# bayesian-A-B-testing

PyMCによるベイズ統計モデリングを用いて、Webマーケティングの広告A/Bテストデータを段階的に分析したプロジェクトです。単純なA/Bテスト(コンバージョン率の群間比較)から出発し、階層モデル・非線形モデリング・モデル診断・ビジネス意思決定への接続まで、一連の分析プロセスを通しで記録しています。

## データ

[Marketing A/B Testing (Kaggle)](https://www.kaggle.com/datasets/faviovaz/marketing-ab-testing)

- 588,101件のユーザーレコード
- `test group`: 広告(`ad`)を見せた群か、公共広告(`psa`)を見せた群か(**ランダム化割り当て**)
- `converted`: 最終的に商品を購入したか(全体で約2.5%)
- `total ads`: そのユーザーが接触した広告の総数(**ランダム化されていない観測変数**)
- `most ads day` / `most ads hour`: 最も多く広告に接触した曜日・時間帯

## 問い

**「広告(ad)はpsaと比べて本当にコンバージョンを増やすのか。また、広告への接触回数はコンバージョンにどう影響するのか、その関係はどこまで信頼して意思決定に使えるのか」**

を、単純な群間比較から始めて、以下のように段階的に掘り下げています。

## 分析の流れ(notebooks/)

| # | ノートブック | 内容 |
|---|---|---|
| 1 | `eda.ipynb` | 基礎的なデータ探索。群間サンプルサイズ、`total ads`の分布(強い右裾)、コンバージョン率などを確認 |
| 2 | `bayes.ipynb` | Beta-Binomialモデルによる最もシンプルなベイズA/Bテスト(ad群 vs psa群のコンバージョン率比較)。Scipyでの解析解とPyMC(MCMC)の両方で実装し、群間差の事後分布・期待損失を評価 |
| 3 | `bayes_logistic.ipynb` | 群間比較に「広告接触回数(`total ads`)」を説明変数として追加したベイズロジスティック回帰(交互作用項あり)。事前予測チェックでJensen不等式に起因する非直感的な挙動を発見し対処。ADVIとNUTSの事後分布を比較し、mean-field ADVIが相関パラメータ間の不確実性を大幅に過小評価すること(SDで約15倍の差)を確認。事後予測チェックで高接触回数帯における外挿の危険性を発見し、「モデルを複雑にする」のではなく「信頼できるデータ範囲を明示して打ち切る」という誠実なアプローチを一度採用 |
| 4 | `cross_random_effects.ipynb` | 曜日(`day`)・時間帯(`hour`)を交差ランダム効果(部分プーリング)として追加した階層ベイズロジスティック回帰。計算コストの都合上、以降はサブサンプル(`df_sub_cross_random_effect_v2.csv`, n≈88,215)を使用 |
| 5 | `evaluation.ipynb` | 階層モデルをLOO/AUC-ROC/Brier scoreで評価。**集計指標では見えない「`log_ads`帯ごとのキャリブレーションのズレ(S字型)」をz値診断で発見**。二次項による対処とその限界(偶関数ゆえに非対称なズレを直せない)、P-スプラインへの切り替えと複数の非識別性の罠(切片との水準の奪い合い、knot配置のトレードオフ、外れ値ダミーとの重複)を、事前予測チェック・感度分析・診断の3点セット(r_hat/ESS/divergence)で一つずつ切り分けて解決 |
| 6 | `spline.ipynb` | `evaluation.ipynb`で設計したP-スプラインモデルの実装・サンプリング |
| 7 | `bayesian_decision_analysis.ipynb` | ここまでの分析結果をビジネス意思決定に接続する最終フェーズ。広告接触回数の限界効果(dp/dx)を可視化し、「変曲点」や「安全域の境界」がデータの疎な領域に出た場合の過適合リスクを診断。C/V比率(コスト/コンバージョン価値)シナリオによる感度分析。**`test group`はランダム化されているため因果効果と言えるが、`total ads`はランダム化されていないため相関関係にとどまる**という因果推論上の限界を明示し、「配信量拡大への投資根拠」ではなく「優良ユーザーの予測的ターゲティング」への活用に着地 |

## 主な結果

- **`test group`(ad vs psa)の効果**: 交互作用項の95%区間が0をまたがず、広告接触回数の効果はad群の方がpsa群より大きい(オッズ比換算でpsa群の約1.4倍の伸び)。`test group`はランダム化されているため、この差は因果効果と解釈できる
- **広告接触回数とCVRの関係は非線形**: 単純な線形ロジスティック回帰では「中間帯で過大評価、両端で過小評価」という系統的なズレ(S字)が生じる。P-スプラインで解消し、LOO(weight=0.98)・AUC-ROC(0.8573)・Brier score(0.0231)のいずれでも最良のモデルとなった
- **上位1%(接触回数の多い層)は外れ値ではなく高価値セグメント**: CVRが通常の6倍(15.4% vs 2.5%)。除外せず、専用のダミー変数で扱った
- **接触回数を増やす投資の根拠には使えない**: `total ads`はランダム化されていない観測変数のため、「配信回数を増やせばCVRが上がる」という因果的主張はデータから導けない。真の因果効果を確認するには、接触頻度自体をランダムに操作する実験が別途必要

詳細な議論の経緯・診断の過程は `notebooks/evaluation.ipynb` と `notebooks/bayesian_decision_analysis.ipynb` を参照してください。

## 得られた方法論的な学び

このプロジェクト全体を通じて繰り返し出てきた原則:

1. **事前予測チェックはサンプリング前に必ず実施する** — Jensen不等式による非直感的な挙動(`beta0`を正しく設定しても、他パラメータの分散が加わるだけで予測平均がズレる)や、パラメータのスケール設定ミス(予測確率が0/1に張り付く)は、この段階でしか気づけないことが多い
2. **r_hat → ESS → divergencesの3段階診断ワークフロー** — 加えて、mean-field ADVIのようにr_hatが構造的に定義できない手法もあるため、手法の特性を理解した上で診断軸を選ぶ
3. **集計指標(LOO, AUC, Brier)だけでモデルの良し悪しを判断しない** — 対数尤度は確率が0付近だと鈍感になるなど、指標ごとの感度特性を理解し、必要なら局所的なキャリブレーション診断(帯ごとのz値など)で裏を取る
4. **モデルを複雑にすると新しい非識別性を生みやすい** — 水準(切片)を複数のパラメータが奪い合う構造にならないよう、役割分担を明示する。事前分布の感度分析(スケールを変えて事後がどれだけ動くか)で「データが支持する値か、事前分布に抑え込まれているだけか」を切り分ける
5. **表面的改善と根本問題の解決を区別する** — target_acceptを上げてdivergencesが減っても、ESSやr_hatが悪化していれば根本解決ではない。事前予測チェックでも、極値(min/max)だけでなく割合や信用区間幅で判断する
6. **外れ値は除外する前に中身を見る** — 統計的な外れ値が、ビジネス上最も価値あるセグメントである可能性がある
7. **意思決定に使う推定は、信用区間が安定している範囲に限定する** — 変曲点やピークが「データの疎な端」に出た場合は、過適合を疑う。モデル改善よりも「信頼できる適用範囲を誠実に明示する」方が実務的に正しい場合がある
8. **相関関係から意思決定を導く際は、変数がランダム化されているかどうかを必ず確認する** — ランダム化された変数と観測変数とでは、結論の強さ(因果 vs 相関)が根本的に異なる
